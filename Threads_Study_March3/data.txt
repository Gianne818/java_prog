To handle specific line ranges in your run() method, you need to track the current line index and use a loop that skips lines until it reaches start, then processes them until it reaches end.

Updated WordCountThread Implementation
Java
@Override
public void run() {
    int count = 0;
    try (BufferedReader br = new BufferedReader(new FileReader(filePath))) {
        String line;
        int currentLine = 0;

        while ((line = br.readLine()) != null) {
            // Only process if the current line is within the assigned range
            if (currentLine >= start && currentLine < end) {
                if (!line.trim().isEmpty()) {
                    String[] words = line.trim().split("\\s+");
                    count += words.length;
                }
            }

            currentLine++;

            // Optimization: Stop reading early if we've passed our range
            if (currentLine >= end) break;
        }
        System.out.println("Thread [" + start + "-" + end + "] Word Count: " + count);
    } catch (Exception e) {
        System.out.println("File Error: " + e.getMessage());
    }
}
Key Logic Changes
currentLine Tracker: A counter is necessary because BufferedReader doesn't inherently know its line number.

Range Condition: The check currentLine >= start && currentLine < end ensures the thread only counts its assigned slice.

Early Break: if (currentLine >= end) break; prevents the thread from reading the rest of the file unnecessarily once its work is done.

Regex Split: Using .split("\\s+") instead of " " is safer, as it handles multiple spaces or tabs between words.

Gemini said
The issue is a classic Race Condition combined with an Offset Logic error.

Why it's returning 0 (or incorrect values)
Race Condition on static totalWordCount: Your threads are all trying to update the same variable at the exact same time. When two threads read totalWordCount, add their local count, and write it back, they overwrite each other's work.

Skipping Lines: BufferedReader doesn't allow you to "jump" to a line. Even if a thread only cares about lines 100–200, it must read through 0–99 first. While your logic handles the range, the br.readLine() must happen for every line until curr reaches start.

The "0" Result: If your threads crash due to file errors or if numLinesPerThread results in 0 (e.g., 5 lines divided by 10 threads), the count stays at zero.

The Fix: Synchronized Updates and Logical Refinement
To fix this, use a synchronized block or an AtomicInteger. Also, ensure the br.readLine() occurs outside the if block so the counter actually increments.

Updated WordCountThread.java
Java
class WordCountThread extends Thread {
    private String filePath;
    private int start, end;
    // Use an Object for a lock to prevent race conditions
    private static final Object lock = new Object();
    static int totalWordCount = 0;

    public WordCountThread(String filePath, int start, int end) {
        this.filePath = filePath;
        this.start = start;
        this.end = end;
    }

    @Override
    public void run() {
        int localCount = 0;
        try (BufferedReader br = new BufferedReader(new FileReader(filePath))) {
            String line;
            int curr = 0;
            while ((line = br.readLine()) != null) {
                // We MUST read every line to reach the 'start' point
                if (curr >= start && curr < end) {
                    String trimmed = line.trim();
                    if (!trimmed.isEmpty()) {
                        // split by whitespace (handles multiple spaces)
                        localCount += trimmed.split("\\s+").length;
                    }
                }
                curr++;
                if (curr >= end) break;
            }

            // Critical Section: Only one thread can update the static variable at a time
            synchronized (lock) {
                totalWordCount += localCount;
            }
        } catch (Exception e) {
            System.err.println("Thread Error: " + e.getMessage());
        }
    }
}

The logic in your run() method is mostly correct for calculating the local count, but you have two specific issues that will keep causing you headaches: the Race Condition on the static variable and the Output Timing.

1. The Race Condition (Why the count might be wrong)
Even though you are using a localCount variable, the final step totalWordCount += localCount; is not atomic.

Behind the scenes, the computer performs three steps:

Read the current value of totalWordCount.

Add localCount to it.

Store the new value back into totalWordCount.

If Thread A and Thread B both "Read" the value at the same time (e.g., both see 100), they will both calculate their own new totals (e.g., 110 and 115) and overwrite each other. One thread's work will be lost. This is likely why you are seeing inconsistent or "0" results.

2. The Print Statement (Why you see "0")
In your code, you print totalWordCount inside the run() method. Because threads run concurrently:

Thread 1 might finish and print before any other thread has even started.

If Thread 1 processed lines 0-5 and found 0 words, it will print "0".

If you want to see the actual final total, you must print it in your Main class after all threads have finished their join() calls.

3. Loop Logic Detail
Your check if (!line.isEmpty()) should check the trimmed version of the line. If a line only contains spaces, .split("\\s+") might still return an array with an empty string, adding 1 to your count incorrectly.

The "Clean" Version
If you want to avoid synchronized but still get a correct answer, the safest way is to avoid the static variable entirely and let Main do the math:

WordCountThread.java

Java
class WordCountThread extends Thread {
    private String filePath;
    int start, end;
    int localCount = 0; // NOT static

    public WordCountThread(String filePath, int start, int end) {
        this.filePath = filePath;
        this.start = start;
        this.end = end;
    }

    @Override
    public void run() {
        try (BufferedReader br = new BufferedReader(new FileReader(filePath))) {
            String line;
            int curr = 0;
            while ((line = br.readLine()) != null) {
                if (curr >= start && curr < end) {
                    String trimmed = line.trim();
                    if (!trimmed.isEmpty()) {
                        localCount += trimmed.split("\\s+").length;
                    }
                }
                curr++;
                if (curr == end) break;
            }
        } catch (Exception e) {
            System.err.println("File Error: " + e.getMessage());
        }
    }
}
